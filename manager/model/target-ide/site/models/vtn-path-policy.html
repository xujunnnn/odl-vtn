RPCs Summary

Name Description
set-path-cost Associate the cost of transmitting a packet with the specified switch port in the specified path policy. This operation takes a list of `vtn-path-cost-config', and put all configurations in that list into the specified path policy. Switch port descriptors specified in the RPC input will be copied to the RPC output, and `status' field in `vtn-port-desc-result' describes the result of the opration. `CREATED' indicates that the link cost configuration for the switch port specified by `port-desc' has been newly created. 'CHANGED' indicates that the link cost configuration for the switch port specified by `port-desc' has been successfully changed. Null indicates that the link cost configuration for the switch port specified by `port-desc' was not changed. On failure, one of vtn-error-tag value which indicates the cause of error is set into application tag in RPC error. `BADREQUEST' is set if the RPC input contains invalid data. `NOTFOUND' is set if the specified path policy is not present. `INTERNALERROR' is set if the operation failed due to internal error.

Values passed to `port-desc' list will be copied to the RPC output, and `status' field in `vtn-port-desc-result' describes the result of the opration. `REMOVED' indicates that the link cost configuration for the switch port associated with the `vtn-port-desc' value has been successfully removed. Null indicates that the link cost configuration for the switch port specified by `port-desc' was not configured in the specified path policy. On failure, one of vtn-error-tag value which indicates the cause of error is set into application tag in RPC error. `BADREQUEST' is set if the RPC input contains invalid data. `NOTFOUND' is set if the specified path policy is not present. `INTERNALERROR' is set if the operation failed due to internal error.

Null indicates that no path policy is present. On failure, one of vtn-error-tag value which indicates the cause of error is set into application tag in RPC error. `INTERNALERROR' is set if the operation failed due to internal error.

If the path policy specified by the path policy ID already exists, it will be modified as specified by the RPC input. `status' in RPC output describes the result of the operation. `CREATED' indicates that the specified path policy has been newly created. `CHANGED' indicates that existing path policy configuration has been successfully changed.       • leaf: id
        • description: The identifier of the path policy.
        • type: int32
      • leaf: default-cost
        • description: The default cost for unspecified link. If the cost of outgoing switch port is not defined in `vtn-path-cost' list, this value is used as the cost. Specifying zero means that the default link cost should be determined individually by the link speed configured to a switch port. If omitted or zero is specified, the default link cost of a switch port is calculated by the following formula: MAX(10,000,000,000,000 / (link speed (bps)), 1)
        • type: default-cost
        • default: 0
      • list: vtn-path-cost
        • description: A list of costs of switch links for transmitting. The cost of using the specific switch port is determined by searching for an element in this list which matches the switch port location. If multiple elements match the target switch port, the VTN Manager gives higher priority to one which specifies more extract port location.

        • uses: vtn-path-cost-config

            • description: Describes the location of the physical switch port.
            • type: (urn:opendaylight:vtn:types)vtn-port-desc
          • leaf: cost
            • description: The cost of using physical switch port. The specified cost is used when a packet is transmitted from the specified switch port. If omitted, it will be treated as if 1 is specified.
            • type: cost
            • default: 1

set-path-cost

  • Description: Associate the cost of transmitting a packet with the specified switch port in the specified path policy. This operation takes a list of `vtn-path-cost-config', and put all configurations in that list into the specified path policy. Switch port descriptors specified in the RPC input will be copied to the RPC output, and `status' field in `vtn-port-desc-result' describes the result of the opration. `CREATED' indicates that the link cost configuration for the switch port specified by `port-desc' has been newly created. 'CHANGED' indicates that the link cost configuration for the switch port specified by `port-desc' has been successfully changed. Null indicates that the link cost configuration for the switch port specified by `port-desc' was not changed. On failure, one of vtn-error-tag value which indicates the cause of error is set into application tag in RPC error. `BADREQUEST' is set if the RPC input contains invalid data. `NOTFOUND' is set if the specified path policy is not present. `INTERNALERROR' is set if the operation failed due to internal error.

          • description: The identifier of the path policy.
          • type: (urn:ietf:params:xml:ns:yang:1)int32
        • list: path-cost-list
          • description: Pairs of the switch port descriptors and link cost configuratins to be applied to the path policy specified by `id'. Note that a duplicate switch port descriptor in this list will cause a `BADREQUEST' error.

Null indicates that existing path policy configuration was not changed. Specifying zero means that the default link cost should be determined individually by the link speed configured to a switch port. If omitted or zero is specified, the default link cost of a switch port is calculated by the following formula: MAX(10,000,000,000,000 / (link speed (bps)), 1)
          • type: default-cost

If multiple elements match the target switch port, the VTN Manager gives higher priority to one which specifies more extract port location.

              • description: Describes the location of the physical switch port.
              • type: (urn:opendaylight:vtn:types)vtn-port-desc
            • leaf: cost
              • description: The cost of using physical switch port. The specified cost is used when a packet is transmitted from the specified switch port. If omitted, it will be treated as if 1 is specified.
              • type: cost
              • default: 1

          • description: Describes how to update the specified path policy if it already exists. If `SET' is specified, the given path policy configuration will be applied as specified.
